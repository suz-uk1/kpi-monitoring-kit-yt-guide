# YouTube週次KPIモニタ（手入力Pro） セットアップガイド

このページは「上から順番にやるだけ」で導入できるように書いています。
（スクショは順次追加します。各手順に「ここをクリック」という目印も書きます）

---

## 0. 全体の流れ（最短5分）
1. テンプレを開く → 自分のDriveにコピーする
2. Apps Script を開く → `main_clean.gs` を貼り付けて保存する
3. スプレッドシートに戻る → `KPI Kit → Setup (create tabs)` を実行（初回だけ）
4. `Input_Weekly` に週1行入力 → `KPI Kit → Update & Send`

---

## 1. テンプレをコピー（自分のDriveに複製）

### 1-1) テンプレを開く
- A. 購入したZIPを解凍します
- B. `01_TEMPLATE/YouTube_KPI_週次モニタ_テンプレ（コピー用URL）.txt` を開きます
- C. 中に書いてある **テンプレコピーURL** をクリックします
- D. ブラウザでスプレッドシートが開けばOKです

（スクショ：テンプレURLを開いた直後の画面）

※このテンプレには見本として **ダミーの入力が1行**入っています。
運用を始めるときは、`Input_Weekly` の2行目（ダミー）を削除してから使ってください。

### 1-2) 自分のDriveにコピーする（ここから運用を始める）
- A. 開いたスプレッドシート上部メニューの **［ファイル］** をクリック
- B. **［コピーを作成］** をクリック
- C. 「コピーを作成」のダイアログで保存先を確認して **［コピーを作成］**
- D. 自分のDriveにコピーされたスプレッドシートが開けばOKです（これが運用版）

（スクショ：ファイル→コピーを作成）

---

## 2. Apps Script を貼り付け（コードを入れる）

### 2-1) Apps Script を開く
- A. 運用版スプレッドシート上部メニューの **［拡張機能］** をクリック
- B. **［Apps Script］** をクリック
- C. Apps Scriptの画面（コードエディタ）が開けばOKです

（スクショ：拡張機能→Apps Script）

### 2-2) `main_clean.gs` を貼り付けて保存
- A. ZIP内の `02_APPS_SCRIPT/main_clean.gs`（または `.gs.txt`）を開く
- B. 中身をすべてコピー（⌘A → ⌘C）
- C. Apps Script側のファイル（例：`コード.gs` / `main_clean.gs`）を開く
- D. 中身を全選択して貼り付け（⌘A → ⌘V）
- E. 上部の保存（⌘S または保存アイコン）で保存

（スクショ：Apps Scriptのエディタで貼り付け）

---

## 3. 初回セットアップ（初回だけ）

### 3-1) スプレッドシートに戻って Setup を実行
- A. ブラウザでスプレッドシートのタブに戻ります
- B. 画面上部のメニューに **`KPI Kit`** が表示されていることを確認
- C. **`KPI Kit → Setup (create tabs)`** をクリック
- D. Home / Input_Weekly / Weekly_Report / Config などが揃えばOKです

（スクショ：KPI Kit → Setup）

### 3-2) 初回の権限承認について（なぜ出る？どう進める？）
初回だけ、Googleから権限承認の画面が出ます。

- このキットは Apps Script で「シートの作成/更新」や「（任意で設定した場合のみ）Discord/Email送信」を行います
- そのためGoogleが「このスクリプトに許可を与えますか？」という確認を出します
- 「安全ではないページ」と出るのは、審査済みの公式アドオンではなく **ユーザーが自分のシートに貼り付けたスクリプト**だからです

承認画面が出たら：
- A. 「詳細」→「（安全ではないページ）に移動」
- B. 権限の内容を確認して **許可**
- C. 許可後、スプレッドシートに戻って **もう一度 `Setup`** を実行

（スクショ：承認画面の例）

※不安な場合は、まず `Config` の Discord/Email を空のまま進めてOKです（空の場合、送信しません）。

---

## 4. 週次の使い方（毎週やること）

### 4-1) 週1行入力
- A. シートタブの **`Input_Weekly`** をクリック
- B. 2行目以降に今週の数字を **1行** 入力
- C. 入力できたらOKです

（スクショ：Input_Weekly）

### 4-2) Update & Send
- A. 上部メニュー **`KPI Kit`** をクリック
- B. **`Update & Send`** をクリック
- C. `Home` が更新され、`Weekly_Report` に週報テキストが出ればOKです

（スクショ：Update & Send → Weekly_Report）

## 5. よくある質問
- `KPI Kit` メニューが出ない：Apps Scriptの貼り付け・保存ができているか確認し、ページ更新→再確認
- 承認後に動かない：`Setup` / `Update & Send` をもう一度

詳しくは同梱の `03_GUIDE/FAQ_TROUBLESHOOT_v2.md` を参照してください。

## 補足：Apps Script画面に出る関数（onOpen / setup / updateAndSend）について
Apps Scriptの画面には関数として `onOpen` / `setup` / `updateAndSend` が表示されることがあります。

- 基本は **スプレッドシート側のメニュー（KPI Kit）から実行**してください
- `onOpen` はメニューを表示するためのものなので、通常は手動実行しません
- もし `KPI Kit` メニューが出ない場合だけ、Apps Script側で `setup` を実行して承認→スプレッドシートを再読み込み、で復旧できます
